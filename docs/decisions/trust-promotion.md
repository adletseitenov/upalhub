# Trust promotion: когда `ai_draft` → `data_refined`?

**Статус: открытый вопрос. Решение: ОЖИДАЕТ основателя.**

## Вопрос

Профили экзаменов (`exam_profiles.trust`) рождаются как `ai_draft` (AI-research).
По какому критерию профиль повышается до `data_refined` (и далее до `verified`)?
Сейчас повышение не реализовано нигде — поле только отображается на странице
профиля.

## Варианты

### A. Повышение после refine создателем

Создатель профиля прогоняет refine (уточнение спеки через `/api/exam-profiles/refine`)
→ профиль автоматически становится `data_refined`.

- (+) Ноль новой инфраструктуры: сигнал уже существует, повышение — одна строка в refine-флоу.
- (+) Понятная семантика: человек посмотрел и поправил данные.
- (−) Один человек — слабая гарантия качества: refine может быть косметическим или даже ухудшающим.
- (−) Создатель не обязательно эксперт по экзамену.

### B. Повышение после N успешных попыток

Профиль становится `data_refined`, когда по его тестам завершено N попыток
(например, 10+) без жалоб/аномалий (доля нулевых результатов, брошенные попытки).

- (+) Сигнал от реального использования, а не от одного человека.
- (+) Автоматически масштабируется без модерации.
- (−) Требует метрик/джоба, которых ещё нет (этап 3+: knowledge_states, аналитика).
- (−) «Успешная попытка» ≠ «корректная спека»: ученики могут проходить тесты по неточному профилю, не замечая этого.

### C. Ручная модерация (админ/эксперт помечает)

Повышение только руками команды (или приглашённого эксперта) после проверки
спеки против официальных источников.

- (+) Максимальное доверие; согласуется с уровнем `verified` как «проверено человеком с авторитетом».
- (+) Простая реализация (SQL/админ-действие), процессная, а не кодовая.
- (−) Не масштабируется на длинный хвост экзаменов; узкое место — время команды.
- (−) На MVP-этапе никто не занимается модерацией системно.

## Возможная комбинация (для обсуждения, не решение)

A → `data_refined` (дёшево, сигнал причастности), C → `verified` (дорого, сигнал
авторитета); B — как телеметрия-подсказка модератору, а не автоматика.

**Решение: ОЖИДАЕТ основателя.**
